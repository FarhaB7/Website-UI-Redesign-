Step 01 — Heuristic Evaluation

10 usability issues identified.

Evaluated against Nielsen's 10 Usability Heuristics. Issues are rated Critical, Major, or Minor based on frequency, impact, and persistence.

● Critical
No system status feedback
Data refreshes with no loading indicator. Users don't know if the system is working, leading to repeated clicks and frustration.
# 1 — Visibility of System Status
● Critical
Inconsistent error messaging
Form validation errors use 3 different styles. Red text, toast messages, and inline alerts appear non-deterministically for the same error type.
# 4 — Consistency & Standards
● Critical
No undo for destructive actions
Deleting a dashboard segment is immediate and irreversible. No confirmation dialog or undo option exists, causing permanent data loss.
# 3 — User Control & Freedom
● Major
Dense information architecture
The primary nav exposes 14 top-level items with no hierarchy. Cognitive load is extreme; users cannot locate features without scanning all options.
# 8 — Aesthetic & Minimalist Design
● Major
Low contrast typography
Body text sits at 3.2:1 contrast ratio — below WCAG AA (4.5:1). Labels on dark chart backgrounds fail entirely at 1.8:1, excluding low-vision users.
# 10 — Help & Documentation
● Major
No recognition of real-world objects
Data export uses jargon like "XLSX_PIPE_EXTRACT" instead of "Download as Excel." Users unfamiliar with internal naming cannot proceed without documentation.
# 2 — Match the Real World
● Minor
Filter state not persisted
Navigating away from a filtered view resets all filters. Users must re-apply date ranges and segment filters on each page visit.
# 3 — User Control & Freedom
● Minor
No keyboard navigation
Dropdown menus and modal dialogs are unreachable via keyboard. Tab order is non-sequential, causing issues for keyboard-only and assistive-technology users.
# 7 — Flexibility & Efficiency
● Minor
Touch targets too small on mobile
Pagination controls and icon buttons measure 18×18px — well below the 44px WCAG recommended minimum — causing frequent mis-taps on touch devices.
# 7 — Flexibility & Efficiency
● Minor
Help content buried in settings
Documentation is 4 clicks deep inside account settings. There is no contextual help or tooltip system for unfamiliar chart types or metrics.

Step 04 — Before & After

A measurable improvement in every dimension.

Before Redesign SUS Score: 34 / 100
❌
14-item flat nav — no grouping or hierarchy, overwhelming cognitive load on every page.
❌
3.2:1 contrast ratio — fails WCAG AA. Key data labels are illegible for 8% of users.
❌
No loading states — silent refreshes leave users clicking repeatedly and losing data.
❌
Mobile unusable — 18px touch targets, horizontal overflow, no responsive layout at all.
❌
Irreversible deletes — no confirmation, no undo. Teams lost weeks of segment work.
System Usability Scale: 34 — Marginally Acceptable
After Redesign SUS Score: 88 / 100
✅
7-item grouped nav — sidebar with clear sections, active states, and keyboard navigation.
✅
7.2:1 contrast ratio — exceeds WCAG AAA. Typography legible for all users in all conditions.
✅
Skeleton loaders + toast system — clear feedback for every async action, users always know the system state.
✅
Mobile-first responsive — 44px touch targets, fluid grid, dedicated mobile nav pattern.
✅
Undo system + confirmation dialogs — 5-second undo toast for all destructive actions, zero accidental data loss.
System Usability Scale: 88 — Excellent
